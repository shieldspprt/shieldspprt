# Cut LLM cost and latency with prompt caching

Long, stable context (system prompts, retrieved docs, tool schemas) is the expensive part of every call. Prompt caching tells the provider to reuse the processed prefix instead of recomputing it, which drops both billable tokens and time to first token on repeat calls.

```ts
// Anthropic Messages API: mark the stable prefix with cache_control.
// Everything before it is hashed once and reused on later calls that share it.
const messages = [
  { role: "user", content: "Summarise the latest audit." },
];

const res = await anthropic.messages.create({
  model: "claude-3-5-sonnet-latest",
  max_tokens: 1024,
  system: [
    {
      type: "text",
      text: BIG_SYSTEM_PROMPT,            // long, rarely changes
      cache_control: { type: "ephemeral" }, // cache from here down
    },
  ],
  messages,
});

// On a cache hit the call returns faster and bills cached-input tokens,
// not full input tokens for the reused prefix.
```

- Put the parts that change least (system prompt, tool definitions, retrieved corpus) at the front and tag the cut point, because caching only pays off when the prefix is shared across calls.
- Cached tokens are billed at a discount and skip prefill, so a chatty app with a fixed persona or a fixed knowledge pack sees the biggest win.
- Mind the minimums and the TTL (inactivity of about 5 minutes evicts the entry), and only one cache breakpoint is free on most plans.
