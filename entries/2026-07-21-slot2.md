Stream partial JSON from LLMs with a tolerant parser so the UI updates token by token.

```typescript
import { z } from "zod";
import { parsePartialJson } from "partial-json";

const ToolCallSchema = z.object({
  name: z.string(),
  arguments: z.record(z.unknown()),
});

function parseStreamingJson(text: string) {
  try {
    return ToolCallSchema.parse(JSON.parse(text));
  } catch {
    const partial = parsePartialJson(text);
    return ToolCallSchema.safeParse(partial);
  }
}

for await (const chunk of stream) {
  const result = parseStreamingJson(accumulated + chunk);
  if (result.success) {
    ui.updateToolCall(result.data);
  }
}
```

- Partial JSON parsing lets you render tool calls and structured fields before the model finishes the full object.
- Validate against the Zod schema at each step so malformed fragments never reach downstream code.
- Keep the parser permissive during streaming; enforce strict validation only on the final complete message.
