Coalesce identical in-flight LLM requests so duplicate prompts share one upstream call and the response fans out to all waiters.

```typescript
const inflight = new Map<string, Promise<LLMResponse>>();

async function cachedComplete(key: string, call: () => Promise<LLMResponse>) {
  const existing = inflight.get(key);
  if (existing) return existing;

  const promise = call().finally(() => inflight.delete(key));
  inflight.set(key, promise);
  return promise;
}

// Usage: deduplicate by prompt hash + model + params
const key = hash(`${prompt}|${model}|${temp}`);
const result = await cachedComplete(key, () => llm.complete(prompt, { model, temp }));
```

- Request coalescing cuts cost and latency when the same prompt arrives in quick succession from multiple clients or retries.
- The key must include all parameters that affect the output (model, temperature, seed, system prompt) to avoid cross-contamination.
- Clear the entry in a finally block so a failed call does not block future attempts with the same key.
