# Deduplicate near-duplicate chunks before embedding to cut RAG cost and retrieval noise

Indexing the same paragraph many times wastes embedding tokens and dilutes retrieval, since duplicate chunks compete for the same top-k slots. Normalize text and drop both exact and near duplicates before you ever call the embedding model.

```ts
type Chunk = { id: string; text: string };

function normalize(text: string): string {
  return text
    .toLowerCase()
    .replace(/\s+/g, " ")
    .replace(/[^\p{L}\p{N} ]/gu, "")
    .trim();
}

function tokens(text: string): Set<string> {
  return new Set(normalize(text).split(" ").filter(Boolean));
}

// Jaccard similarity of token sets
function nearDuplicate(a: Set<string>, b: Set<string>): boolean {
  if (a.size === 0 || b.size === 0) return false;
  let inter = 0;
  for (const t of a) if (b.has(t)) inter++;
  const jaccard = inter / (a.size + b.size - inter);
  return jaccard > 0.85;
}

export function dedupe(chunks: Chunk[]): Chunk[] {
  const seenText = new Map<string, string>(); // norm -> id
  const kept: Chunk[] = [];
  const keptTokens: Set<string>[] = [];

  for (const c of chunks) {
    const norm = normalize(c.text);
    if (!norm) continue;
    if (seenText.has(norm)) continue; // exact duplicate
    const tok = tokens(c.text);
    if (keptTokens.some((k) => nearDuplicate(k, tok))) continue; // near duplicate
    seenText.set(norm, c.id);
    keptTokens.push(tok);
    kept.push(c);
  }
  return kept;
}
```

- Exact-dup removal by normalized hash is nearly free and catches most repeats created by overlapping sliding windows.
- Near-dup via token Jaccard at a 0.85 threshold trims boilerplate (headers, footers, legal lines) that would otherwise crowd the top-k results.
- Fewer, cleaner chunks mean fewer embedding calls and a sharper context window for the LLM at query time.
