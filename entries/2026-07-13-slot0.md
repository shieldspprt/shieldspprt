# Make agent tool calls idempotent with a deterministic key

When an LLM agent retries after a timeout, naive tool execution can double-book a flight or double-charge a card. Derive an idempotency key from the tool name plus normalized arguments, claim it in a store before you act, and return the cached result if the key is already in flight.

```ts
// Hono route that wraps a risky agent tool behind an idempotency guard.
const TOOLS = {
  charge_card: async (args: { userId: string; cents: number }) => { /* ... */ },
};

export async function runTool(c: Context) {
  const body = await c.req.json<{ tool: string; args: unknown; nonce?: string }>();
  const { tool, args, nonce } = body;

  // Normalise so key order and whitespace never change the hash.
  const canon = crypto
    .createHash("sha256")
    .update(JSON.stringify({ tool, args }))
    .digest("hex");

  // Prefer the caller's nonce when present, else fall back to the body hash.
  const key = nonce ? `${tool}:${nonce}` : `sig:${canon}`;

  const redis = c.get("redis");
  const INFLIGHT = 30; // seconds; longer than the tool's normal runtime

  // SET NX: first caller wins, everyone else waits or reads the result.
  const owner = await redis.set(`tool:${key}`, "PENDING", "EX", INFLIGHT, "NX");
  if (!owner) {
    const prior = await redis.get(`result:${key}`);
    if (prior) return c.json(JSON.parse(prior));
    await sleep(250); // brief backoff, then retry the claim
    return runTool(c);
  }

  const result = await TOOLS[tool as keyof typeof TOOLS](args as any);
  await redis.set(`result:${key}`, JSON.stringify(result), "EX", 3600);
  return c.json(result);
}
```

- The key is built from the tool plus arguments, not from a timestamp, so identical requests collide intentionally and only the first one executes.
- `SET key value EX t NX` is atomic, so a concurrent retry either becomes the owner or sees the in-flight marker, and no second execution happens.
- Store the completed result and read it on replay, which turns a flaky retry into a clean, observable no-op for the caller.
