Bound the number of concurrent tool calls an agent can fire in a single turn. Without a limit, a model that decides to call ten tools at once can exhaust rate limits, spike latency, and make the UI feel broken. A tiny semaphore around the tool executor keeps the fan-out predictable and the user experience smooth.

```ts
const toolSemaphore = new Semaphore(3)

async function executeToolCall(call: ToolCall) {
  return toolSemaphore.runExclusive(async () => {
    const result = await toolRegistry.execute(call.name, call.args)
    return { callId: call.id, result }
  })
}

const results = await Promise.all(
  toolCalls.map(call => executeToolCall(call))
)
```

- A semaphore of 3 to 5 covers most real workloads without starving the event loop.
- Pair it with a per-turn token budget so a chatty model cannot burn the quota on tool calls alone.
- Log the wait time per call; if it trends up, raise the limit or add a queue-depth metric.
