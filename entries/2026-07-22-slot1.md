Validate structured LLM output with a strict schema before it reaches your database. A malformed response can cascade into constraint violations, corrupted rows, or silent data loss downstream.

```python
from pydantic import BaseModel, ValidationError
from typing import Literal

class Extraction(BaseModel):
    entity: str
    type: Literal["person", "org", "location", "date"]
    confidence: float

def safe_parse(raw: str) -> Extraction | None:
    try:
        return Extraction.model_validate_json(raw)
    except ValidationError as e:
        log.warning("llm output failed schema", errors=e.errors())
        return None

result = safe_parse(llm_response)
if result:
    db.insert(result.model_dump())
```

- Treat the LLM as an untrusted client. Validate every field before persistence
- Use a strict schema with literal enums and constrained numerics to catch hallucinated values early
- Log validation failures with the raw response for prompt iteration, do not silently drop them
