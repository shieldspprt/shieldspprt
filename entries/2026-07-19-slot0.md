# Keep your RAG retriever honest with a score floor

A vector search that returns the top-k nearest chunks will happily hand your LLM irrelevant context when nothing in the corpus actually matches the question. Add a minimum similarity score and a fallback message so the model never answers from noise.

```python
import numpy as np
from openai import OpenAI

client = OpenAI()

# cosine similarity between two unit vectors
def sim(a: np.ndarray, b: np.ndarray) -> float:
    return float(np.dot(a, b))

SCORE_FLOOR = 0.78  # tuned per corpus; raise to be stricter

def retrieve(question: str, chunks: list[dict], k: int = 4):
    q = client.embeddings.create(model="text-embedding-3-small",
                                 input=question).data[0].embedding
    scored = []
    for c in chunks:
        s = sim(np.array(q), np.array(c["embedding"]))
        if s >= SCORE_FLOOR:
            scored.append((s, c))
    scored.sort(key=lambda x: x[0], reverse=True)
    return [c for _, c in scored[:k]]

def answer(question: str, chunks: list[dict]):
    hits = retrieve(question, chunks)
    if not hits:
        return "I could not find anything relevant in the docs for that."
    context = "\n\n".join(h["text"] for h in hits)
    # send context to the LLM as usual...
    return context
```

- A fixed top-k hides the difference between "four strong matches" and "four weak ones." A score floor surfaces that gap.
- Set the floor from a held-out question set, not a guess. Too low and noise leaks in, too high and you reject real answers.
- When nothing clears the floor, return a no-match message rather than stuffing weak context. The retriever being honest beats it being chatty.
