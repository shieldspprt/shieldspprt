# Constrain LLM output with a JSON schema instead of parsing prose

When you need structured data from a model, ask for it with a strict JSON schema rather than prompting it to "return JSON." A schema gives you a guaranteed shape, typed fields, and no fragile string parsing.

```typescript
// Bad: prompt for JSON, then hope the model obeyed and parse it.
const res = await openai.chat.completions.create({
  model: "gpt-4o-mini",
  messages: [{ role: "user", content: "Extract name, age, city from: " + text }],
});
const parsed = JSON.parse(res.choices[0].message.content!); // throws on "Sure! {...}"

// Good: enforce the shape server-side with strict json_schema.
const res = await openai.chat.completions.create({
  model: "gpt-4o-mini",
  messages: [{ role: "user", content: "Extract the person from: " + text }],
  response_format: {
    type: "json_schema",
    json_schema: {
      name: "person",
      strict: true,
      schema: {
        type: "object",
        properties: {
          name: { type: "string" },
          age: { type: "integer" },
          city: { type: "string" },
        },
        required: ["name", "age", "city"],
        additionalProperties: false,
      },
    },
  },
});
const person = JSON.parse(res.choices[0].message.content!); // shape is guaranteed
```

- `strict: true` forces the model to fill every `required` field, so you get a complete object instead of partial prose.
- Validate even schema output at the boundary. Treat the model as an untrusted source and run the parsed object through your own validator before it touches a database.
- Keep the schema small and flat. Nested, huge schemas push the model toward mistakes and inflate token cost on every call.
