Use a request-scoped cache to deduplicate identical LLM calls within a single user request. When multiple agents or tools hit the same prompt in one turn, a small in-memory map keyed by the normalized request avoids redundant upstream calls and keeps latency predictable.

```typescript
const requestCache = new Map<string, Promise<LLMResponse>>()

async function cachedLLMCall(params: LLMParams): Promise<LLMResponse> {
  const key = JSON.stringify(params)
  if (requestCache.has(key)) {
    return requestCache.get(key)!
  }
  const promise = llmClient.call(params)
  requestCache.set(key, promise)
  try {
    return await promise
  } finally {
    requestCache.delete(key)
  }
}
```

- Prevents double-billing and double-latency when two tools request the same completion
- The `finally` block ensures the cache entry is removed even if the call fails, so retries work
- Scope the cache to the request lifetime, not the process, to avoid stale results across users
