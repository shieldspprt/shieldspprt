# Structured Output Validation with Pydantic and Retry Logic

When calling LLMs for structured output, wrap the call in a retry loop that validates against a Pydantic model and re-prompts on failure. This avoids silent schema drift and keeps pipelines reliable.

```python
from pydantic import BaseModel, ValidationError
from openai import OpenAI
from tenacity import retry, stop_after_attempt, retry_if_exception_type

class Extraction(BaseModel):
    entities: list[str]
    sentiment: str
    confidence: float

client = OpenAI()

@retry(
    retry=retry_if_exception_type(ValidationError),
    stop=stop_after_attempt(3),
    reraise=True,
)
def extract_with_retry(text: str) -> Extraction:
    resp = client.beta.chat.completions.parse(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": f"Extract: {text}"}],
        response_format=Extraction,
    )
    return resp.choices[0].message.parsed

result = extract_with_retry("The launch was delayed but the team stayed calm.")
```

- Validate every LLM structured response against a schema before it reaches downstream code
- Retry with the validation error fed back to the model so it can self-correct
- Cap retries at 2-3 attempts to bound latency and cost
