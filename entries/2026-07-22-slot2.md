Prefer streaming LLM responses over HTTP chunked transfer instead of buffering the full response. It reduces perceived latency and lets the client render tokens as they arrive.

```python
from fastapi import StreamingResponse
from openai import AsyncOpenAI

client = AsyncOpenAI()

async def stream_completion(prompt: str):
    async def gen():
        stream = await client.chat.completions.create(
            model="gpt-4o-mini",
            messages=[{"role": "user", "content": prompt}],
            stream=True,
        )
        async for chunk in stream:
            delta = chunk.choices[0].delta.content or ""
            if delta:
                yield f"data: {delta}\n\n"
        yield "data: [DONE]\n\n"
    return StreamingResponse(gen(), media_type="text/event-stream")
```

- Stream tokens directly to the client via Server-Sent Events instead of buffering
- Set `media_type="text/event-stream"` and yield `data: <token>\n\n` frames
- Handle client disconnects by catching `asyncio.CancelledError` in the generator
