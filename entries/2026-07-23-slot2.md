Rate-limit LLM calls per user with a token bucket

A token bucket gives you smooth per-user rate limits without hard cutoffs. Each user gets a bucket of tokens that refills at a fixed rate. An LLM call consumes tokens (prompt + estimated completion). If the bucket is empty, the request waits or fails fast. This prevents one user from starving others and makes costs predictable.

```python
import time
import threading
from dataclasses import dataclass, field

@dataclass
class TokenBucket:
    capacity: int
    refill_rate: float  # tokens per second
    tokens: float = field(init=False)
    last_refill: float = field(init=False)
    lock: threading.Lock = field(default_factory=threading.Lock, init=False)

    def __post_init__(self):
        self.tokens = float(self.capacity)
        self.last_refill = time.monotonic()

    def _refill(self):
        now = time.monotonic()
        elapsed = now - self.last_refill
        self.tokens = min(self.capacity, self.tokens + elapsed * self.refill_rate)
        self.last_refill = now

    def take(self, cost: int) -> bool:
        with self.lock:
            self._refill()
            if self.tokens >= cost:
                self.tokens -= cost
                return True
            return False

    def wait_for(self, cost: int, timeout: float = 30.0) -> bool:
        deadline = time.monotonic() + timeout
        while time.monotonic() < deadline:
            if self.take(cost):
                return True
            time.sleep(0.05)
        return False

# Per-user buckets stored in Redis or in-memory dict
buckets: dict[str, TokenBucket] = {}

def get_bucket(user_id: str) -> TokenBucket:
    if user_id not in buckets:
        buckets[user_id] = TokenBucket(capacity=100_000, refill_rate=2_000)  # ~2k tokens/s
    return buckets[user_id]

def estimate_tokens(prompt: str, max_completion: int = 4096) -> int:
    return len(prompt) // 4 + max_completion

def call_llm_with_rate_limit(user_id: str, prompt: str) -> str:
    bucket = get_bucket(user_id)
    cost = estimate_tokens(prompt)
    if not bucket.wait_for(cost, timeout=10.0):
        raise RuntimeError("Rate limit exceeded, try again later")
    # ... call your LLM provider ...
    return "response"
```

- Bucket capacity equals your per-user daily budget. Refill rate spreads it across the day.
- Estimate tokens before the call (prompt length / 4 + max_completion) so you deduct upfront.
- Use Redis with a Lua script for distributed deployments so the check-and-deduct is atomic.
- Return 429 with `Retry-After` when the bucket is empty so clients back off cleanly.
