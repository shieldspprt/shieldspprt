Validate structured LLM output with Zod schemas at the tool boundary, not in the prompt.

```typescript
import { z } from "zod";
import { tool } from "ai";

const ExtractWalletHealthSchema = z.object({
  address: z.string().min(32).max(44),
  solBalance: z.number().nonnegative(),
  tokenAccounts: z.number().int().nonnegative(),
  riskyApprovals: z.array(z.object({
    programId: z.string(),
    allowance: z.string(),
    riskLevel: z.enum(["low", "medium", "high"]),
  })),
  rentExempt: z.boolean(),
});

export const walletHealthTool = tool({
  description: "Fetch and validate wallet health metrics",
  parameters: ExtractWalletHealthSchema,
  execute: async ({ address }) => {
    const raw = await fetchWalletData(address);
    return ExtractWalletHealthSchema.parse(raw);
  },
});
```

- Zod parsing at the tool boundary catches schema drift before it reaches your agent loop.
- The schema doubles as the tool parameter definition, so the LLM and your code share one source of truth.
- Parse failures become structured tool errors the model can retry with corrected arguments.
