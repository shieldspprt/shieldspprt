# Gzip your server-sent-event stream, or you pay 10x for tokens you never read

SSE responses are plain text, and LLM token streams are extremely repetitive. Enabling gzip on the stream cuts bytes over the wire by roughly 8 to 10x, which matters the moment you serve more than a handful of concurrent generations.

```ts
// Bun/Hono: compress the SSE body so token streams are not sent raw
import { Hono } from "hono";
import { compress } from "hono/compress";

const app = new Hono();
app.use("/api/stream", compress({ encoding: "gzip" }));

app.get("/api/stream", async (c) => {
  const stream = new ReadableStream({
    async start(controller) {
      for await (const token of generateTokens()) {
        controller.enqueue(`data: ${JSON.stringify({ token })}\n\n`);
      }
      controller.enqueue("data: [DONE]\n\n");
      controller.close();
    },
  });
  return new Response(stream, {
    headers: {
      "content-type": "text/event-stream",
      "cache-control": "no-cache",
      "x-accel-buffering": "no", // disable proxy buffering on nginx
    },
  });
});
```

- Streaming text compresses well because adjacent tokens repeat shared prefixes and stop words, so gzip ratios stay high end to end.
- Set `x-accel-buffering: no` so a reverse proxy forwards chunks immediately instead of buffering the whole response.
- Compress at the edge or framework layer, not per token, so you are not paying header overhead on every flush.
