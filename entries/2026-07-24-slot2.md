# Circuit Breaker for LLM Provider Failures

Wrap every LLM provider call with a circuit breaker so a single degraded endpoint does not cascade into a full outage. Track success and failure counts in a sliding window; trip the breaker after a threshold (for example 5 failures in 10 seconds) and fail fast with a cached response or a clear error. After a cooldown period, allow a single probe request through; if it succeeds, close the breaker. This pattern keeps the rest of your system responsive while the provider recovers and prevents retry storms from amplifying the outage.

```python
import time
import threading
from dataclasses import dataclass
from typing import Callable, TypeVar, Generic

T = TypeVar("T")

@dataclass
class CircuitBreaker(Generic[T]):
    failure_threshold: int = 5
    window_seconds: int = 10
    cooldown_seconds: int = 30
    _failures: list[float] = None
    _state: str = "closed"  # closed, open, half-open
    _opened_at: float = 0
    _lock: threading.Lock = None

    def __post_init__(self):
        self._failures = []
        self._lock = threading.Lock()

    def call(self, fn: Callable[[], T]) -> T:
        with self._lock:
            now = time.time()
            self._failures = [t for t in self._failures if now - t < self.window_seconds]

            if self._state == "open":
                if now - self._opened_at >= self.cooldown_seconds:
                    self._state = "half-open"
                else:
                    raise CircuitOpenError("circuit breaker open")

        try:
            result = fn()
        except Exception as e:
            with self._lock:
                self._failures.append(now)
                if len(self._failures) >= self.failure_threshold:
                    self._state = "open"
                    self._opened_at = now
            raise

        with self._lock:
            if self._state == "half-open":
                self._state = "closed"
                self._failures.clear()
        return result

class CircuitOpenError(Exception):
    pass
```

- Trip the breaker on a sliding-window failure count, not a raw total, so brief blips do not trigger it
- Half-open state lets a single probe through; success closes the circuit, failure reopens it
- Pair with a fallback (cached response, degraded mode) so the caller never propagates gracefully to the user
