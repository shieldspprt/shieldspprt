# Token budgets keep agent loops from spiraling

A hard token cap per agent turn prevents runaway context growth and cost surprises. Track cumulative tokens in the loop state, enforce a ceiling, and summarise or prune history when the cap is reached. This is the single most effective guardrail for multi-turn agents.

```typescript
interface AgentState {
  history: Message[];
  totalTokens: number;
  readonly MAX_TOKENS = 128_000;
}

function addMessage(state: AgentState, msg: Message): AgentState {
  const newTotal = state.totalTokens + countTokens(msg);
  if (newTotal > state.MAX_TOKENS) {
    const summarised = summariseHistory(state.history);
    return {
      history: [summarised, msg],
      totalTokens: countTokens(summarised) + countTokens(msg),
      MAX_TOKENS: state.MAX_TOKENS,
    };
  }
  return {
    history: [...state.history, msg],
    totalTokens: newTotal,
    MAX_TOKENS: state.MAX_TOKENS,
  };
}
```

- Track cumulative tokens in loop state, not just the last call.
- Summarise or drop oldest turns when the budget is exceeded.
- A hard cap is cheaper and more predictable than dynamic truncation.
