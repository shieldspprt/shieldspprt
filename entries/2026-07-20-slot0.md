Stream tool-call results to the client instead of buffering the full response. When an agent calls multiple tools, sending each result as a server-sent event keeps the UI responsive and lets the user see progress in real time.

```tsx
async function* streamAgentRun(input: string) {
  const stream = await fetch("/api/agent/run", {
    method: "POST",
    body: JSON.stringify({ input }),
    headers: { "Content-Type": "application/json" },
  });

  for await (const chunk of stream.body?.getReader() ?? []) {
    const text = new TextDecoder().decode(chunk);
    for (const line of text.split("\n")) {
      if (line.startsWith("data: ")) {
        yield JSON.parse(line.slice(6));
      }
    }
  }
}

// Client usage
for await (const event of streamAgentRun("Analyze this PR")) {
  if (event.type === "tool_start") showSpinner(event.tool);
  if (event.type === "tool_result") renderResult(event.tool, event.output);
  if (event.type === "final") setAnswer(event.text);
}
```

- Streaming each tool result avoids the "spinner of death" while the agent thinks
- SSE works over standard HTTP and plays nice with load balancers
- The client can render partial UI (tool name, args, partial output) before the final answer arrives
