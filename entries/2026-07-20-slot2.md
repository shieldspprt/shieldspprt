Eval-driven prompt design beats prompt engineering by feel. Write a small eval harness that scores outputs against golden cases then iterate prompts until the score plateaus.

```ts
interface Case { input: string; expected: string }
async function evalPrompt(prompt: string, cases: Case[]) {
  let pass = 0
  for (const c of cases) {
    const out = await llm({ prompt, input: c.input })
    if (grade(out, c.expected)) pass++
  }
  return pass / cases.length
}

// Iterate: change prompt -> run eval -> keep if score improves
```

- A 20-case eval catches regressions that vibe-checking misses
- Store golden cases as JSON so CI can gate prompt changes
- Start with exact-match grading, upgrade to LLM-as-judge only when needed
