Semantic caching cuts LLM costs for near-duplicate prompts

Embed the user prompt, check a vector store for close neighbors, and return the cached completion if cosine similarity exceeds a threshold. Store the prompt embedding with the response at write time. A 0.95 threshold catches paraphrases without false hits.

```python
import numpy as np
from sentence_transformers import SentenceTransformer

model = SentenceTransformer("sentence-transformers/all-MiniLM-L6-v2")
CACHE = {}  # replace with Redis/Postgres vector index in prod

def get_or_compute(prompt: str, llm, threshold: float = 0.95):
    emb = model.encode(prompt, normalize_embeddings=True)
    for cached_emb, cached_resp in CACHE.values():
        if np.dot(emb, cached_emb) >= threshold:
            return cached_resp, True
    resp = llm(prompt)
    CACHE[prompt] = (emb, resp)
    return resp, False
```

- Treat the cache as probabilistic: a near-duplicate hit saves the full generation cost; a miss falls back cleanly
- TTL the entries or cap the store size so stale patterns do not accumulate
- Log hit rate and threshold tuning separately from your main eval harness
